# prompt.md｜今日食籤 — 減脂減醣菜單抽籤工具 完整規格

> 將本文件連同菜單資料（`meal-lottery.jsx` 中的 `RAW` 常數，或三張原始菜單圖片）一併提供給 AI，即可重建或延伸本專案。菜單內容為付費客製、非公開資訊，重建時不得加入任何營養師品牌名稱。

---

## 1. 專案概述

- **名稱**：今日食籤
- **目的**：使用者依照營養師開立的三份減脂減醣外食菜單（1600／1800／2000 大卡）進食。本工具解決「今天吃什麼」的選擇困難：以抽籤方式從菜單中隨機抽出餐點，並可瀏覽、搜尋完整菜單。
- **使用情境**：主要在手機瀏覽器使用（加入主畫面當 App），桌面瀏覽器亦需正常。
- **部署**：GitHub Pages 靜態託管，單一 `index.html`。也需支援本機直接雙擊開啟（`file://`）與完全離線使用。

## 2. 資料規格

### 2.1 結構

菜單資料以「完全對應原始菜單版面」的結構保存，不可扁平化，因為表格模式要照原樣呈現、抽籤池由它展開：

```
RAW = {
  1600: [ 第1組, 第2組, 第3組 ],   // 每個熱量固定 3 組菜單表
  1800: [ ... ],
  2000: [ ... ],
}
每組 = { 餐別名: [ 組合一, 組合二, 組合三, 組合四, 組合五 ] }   // 每餐別固定 5 欄
每個組合 = { store: "店家名" | null, items: ["品項1", "品項2", ...] }
```

- 餐別順序 `MEAL_ORDER`：1600 與 1800 為「早餐、午餐、晚餐、點心」；**2000 為「早餐、午餐、午後點心、晚餐、晚間點心」（一天兩次點心）**。
- 點心類多數無店家，`store: null`（水果、無糖豆漿等通路皆可取得者）。
- 六大類每日份數 `SERVINGS`（隨熱量顯示）：
  - 1600：澱粉 8~9份、豆魚蛋肉 8~10份、油脂 4~6份、蔬菜 4份、水果 1份、乳品 1份
  - 1800：澱粉 9~10份、豆魚蛋肉 9~10份、油脂 6份、蔬菜 6份、水果 1份、乳品 1份
  - 2000：澱粉 12份、豆魚蛋肉 12份、油脂 6份、蔬菜 6份、水果 1份、乳品 1份

### 2.2 衍生資料

- **抽籤池**：每個熱量、每個餐別，將 3 組 × 5 欄展開成池；**以「店家+品項」為 key 去重**（原始菜單有少數重複格，表格照原樣顯示兩格，但抽籤機率不因此提高）。
- **一日組合池**：每個熱量 3 組 × 5 欄 = 15 個完整一日組合（三個熱量共 45 組）。每欄是營養師配好份數的整套（同欄的早/午/晚/點心屬於一組）。

## 3. 功能需求

頂部提供兩個檢視切換：**🎋 抽籤模式** 與 **📋 表格模式**。熱量選擇（1600／1800／2000 大卡按鈕 + 六大類份數標籤）為兩模式共用。

### 3.1 抽籤模式

- **抽的範圍**：「一整天」或單一餐別（餐別按鈕依當前熱量的 MEAL_ORDER 動態產生）。
- **一整天有兩種抽法**：
  - **照組合抽（預設）**：從該熱量 15 個一日組合中抽一整欄，抽出後顯示來源徽章（例：「第2組・組合三」），並提示可到表格模式對照。連抽時避免抽到與上一次相同的組合。
  - **自由混搭**：各餐別從各自的池獨立抽。
- **抽籤按鈕**：按下後有短暫搖晃動畫再揭示結果；顯示累計「第 N 抽」。
- **結果呈現**：每餐一張直式「籤卡」——紅色頂帶（餐別圖示＋「早餐籤」等字樣）、店家名（襯線大字）、品項逐行以虛線分隔。卡片進場有交錯的上滑動畫。
- **重抽單一籤**：每張籤卡有「重抽這一籤」；在照組合抽的結果上重抽任一餐後，組合徽章消失（不再是原始配組）。重抽避免連續抽到同一選項。
- **指定商家篩選**：
  - 商家籤片列，依該熱量選項數量降冪排列，每片顯示「店名 + 數量」；可複選，不選 = 不限。
  - 勾選後顯示即時摘要：「可抽選項：早餐 3・午餐 2⋯」；若某餐別為 0，附提示「多勾幾家」與「清除」按鈕。
  - `store: null` 的通用點心**不受篩選影響**，永遠在池中。
  - 篩選啟用時，「照組合抽」不可用（單一商家湊不齊整欄），一整天自動改自由混搭並顯示說明、照組合抽按鈕呈半透明。
  - 某餐別池為空時抽出「抽不出來」灰色籤卡（頂帶灰色、說明文字引導放寬篩選），不可無聲跳過。
  - 切換熱量時重置商家篩選（各熱量商家清單不同）。
- **輸出今日籤圖片**：
  - 有結果時顯示「🖼️ 輸出今日籤圖片」按鈕。
  - 以**純 Canvas 繪製**（禁用 html2canvas 等外部函式庫）產生直式 PNG（寬 1080，高度依內容自適應）：標題「今日食籤」、日期（YYYY 年 M 月 D 日（週X））、熱量徽章（含組合來源，若為照組合抽）、各餐區塊（紅色餐別帶＋店家襯線大字＋品項列表、餐間虛線分隔）、底部中性標註「菜單由營養師規劃・減脂減醣」、外框圓角描邊、米黃漸層底。品項過長需自動換行（逐字量測寬度）。
  - 繪製前等待 `document.fonts.ready`（失敗則退回系統字體）。
  - **不可直接以 `data:` URL 觸發下載**（iOS Safari 會顯示醜陋的檔案頁）。改為：`canvas.toBlob` → 頁面內彈出預覽 modal，內含圖片預覽（可長按儲存）與三個按鈕——「分享／存到相簿」（`navigator.canShare({files})` 成立時呼叫 `navigator.share`，否則退回下載）、「下載檔案」（blob object URL + `download` 屬性，檔名 `今日食籤_YYYYMMDD.png`）、「關閉」。點遮罩也可關閉；關閉時 `revokeObjectURL`。
  - 匯出只包含當前顯示的餐別（單抽單餐則只輸出那一餐），空籤（抽不出來）不納入。

### 3.2 表格模式

- 依原始菜單版面呈現：當前熱量的 3 組菜單表，每組一張卡（深色標題列「第 N 組菜單」），表格欄為組合一～五、列為餐別。格內店家紅色粗體、品項逐行。
- 手機上表格可橫向捲動，餐別欄 `position: sticky` 固定在左側。
- **關鍵字搜尋**（置頂）：
  - 即時搜尋，**同時跨三個熱量**比對店家與品項（不分大小寫）。
  - 結果列表每筆標示位置「1800卡・第1組・組合一・晚餐」＋內容摘要，關鍵字以 `<mark>` 黃底標亮；點擊其他熱量的結果會切換至該熱量。列表過長可捲動。
  - 當前熱量表格中符合的格子整格標亮（底色 + 內框），格內關鍵字同樣標亮。
  - 查無結果顯示友善提示。
  - 搜尋不受餐別篩選影響（維持全域，避免漏找）。
- **餐別篩選**（搜尋框下方）：「全部」＋各餐別籤片（含圖示），可複選，只顯示勾選的列。2000 卡自動出現五種餐別。切換熱量時重置。

### 3.3 通用行為

- 切換熱量／餐別／抽法／商家篩選時清空目前抽籤結果。
- 單一餐別模式在切換熱量後若該餐別不存在（如 2000 → 1600 的「午後點心」），自動退回「一整天」。

## 4. 視覺風格

台式「求籤／籤詩」美學，溫暖米黃紙感 + 廟宇朱紅：

- **色票**：墨 `#2B2320`（主文字/邊框）、朱 `#C8402A`（主色，深階 `#A93320`）、紙 `#FBF3E2`（頁底漸層搭 `#FFF9EC`→`#F6E7C9`）、蜜 `#E8A33D`（focus 外框/標亮）、青 `#4A7C59`（重抽按鈕）。卡片底 `#FFFBF0`，淡邊 `#E4D3B4`，虛線 `#DECBA6`。
- **字體**：標題與店家名用 `Noto Serif TC`（700/900），內文 `Noto Sans TC`，Google Fonts 載入、離線退回 `PingFang TC / Microsoft JhengHei`。標題「今日食籤」大字寬字距。
- **元件語彙**：圓角膠囊按鈕；選中態為實色填滿（熱量鈕填朱色、餐別鈕填墨色）；卡片 2px 墨色描邊 + 偏移硬陰影（`4px 6px 0 rgba(43,35,32,.16)` 風格）；籤卡底部大圓角模擬籤條。抽籤主按鈕為朱紅漸層膠囊、寬字距襯線字、含 🎋。
- **動效**：抽籤按鈕搖晃 ~0.55s；籤卡上滑進場、每張延遲 90ms、帶輕微回彈曲線；hover 微位移。**必須尊重 `prefers-reduced-motion`**（停用所有動畫）。
- **無障礙**：`button:focus-visible` 蜜色外框；檢視切換使用 `role="tablist"/tab`；搜尋框有 `aria-label`。
- RWD：整體置中、內容區最大寬約 640px（控制區）／1080px（表格與結果區），手機單欄自然流動。

## 5. 技術限制與交付

- **技術棧**：React 18（僅 useState/useEffect/useMemo）、inline style 物件 + 一段全域 CSS 字串。**禁用 localStorage/sessionStorage**（狀態一律 in-memory）。不使用任何第三方 UI/繪圖/下載函式庫。
- **交付兩個檔案**：
  1. `meal-lottery.jsx` — 原始碼，單一元件 default export，菜單資料內含。
  2. `index.html` — 部署檔：jsx 以 Babel（`@babel/preset-react`，**classic runtime、development: false**）編譯後，與 React 18 / ReactDOM 18 的 **UMD production 版原始碼一併內嵌**於同一 HTML（不可依賴 unpkg 等 CDN，`file://` 離線開啟必須可用）。`<head>` 含 viewport、theme-color `#C8402A`、emoji 🎋 的 data-URI favicon、標題「今日食籤｜減脂減醣菜單抽籤」。
- 內嵌前檢查程式碼中不得出現 `</script` 字串以免提前終止 script 區塊。
- README 與畫面文字**一律使用中性描述「菜單由營養師規劃」**，不得出現營養師品牌名稱或原始圖文連結。

## 6. 驗收要點（含既知測試陷阱）

- 功能驗收：三熱量切換與份數顯示；照組合抽出的四（五）餐與表格同欄一致；自由混搭與單餐抽；商家篩選的計數、空餐別灰籤、清除、與照組合抽互斥；重抽不連續重複；匯出圖片 modal 的分享/下載/關閉；表格搜尋跨熱量、點擊跳轉、標亮；表格餐別篩選列數正確（1600/1800 全部 12 列、2000 全部 15 列）。
- 若以 jsdom 做自動化驗證，注意：
  - 斷言文字請查 `#root` 的 textContent，**不可查 `body`**（內嵌 script 原始碼也在 body 內，會誤判）。
  - 觸發互動用元素原生 `.click()`；若用 `dispatchEvent` 必須帶 `cancelable: true`，否則 React 18 可能忽略。
  - jsdom 未實作 `download` 屬性與 canvas，需 stub `HTMLAnchorElement.prototype.click`、`getContext`（mock `measureText` 等）、`toBlob`、`URL.createObjectURL/revokeObjectURL`。
